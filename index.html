Internet merupakan jaringan luas yang menghubungkan komputer di seluruh dunia. Komputer di Internet terhubung secara fisik dan menggunakan bahasa (protokol) yang sama untuk berkomunikasi.

Arsitektur WebGIS

  1. Client adalah perangkat yang melakukan akses terhadap website (orang/manusia yang mengakses juga dapat disebut client). Client berkomunikasi dengan server melalui web protokol seperti HTTP (Hyper Text Transfer Protocol).

  2. Untuk menampilkan dan berinteraksi dengan data GIS, sebuah browser membutuhkan Plug-In atau Java Applet atau bahkan keduanya. Hal ini dikarenakan web browser standar tidak memiliki kemampuan untuk menampilkan data GIS.

  3. Web server bertanggung jawab terhadap proses permintaan dari client dan mengirimkan tanggapan terhadap respon tersebut. Dalam arsitektur web, sebuah web server juga mengatur komunikasi dengan komponen server side GIS.

  4. Komponen Server side GIS bertanggung jawab terhadap koneksi kepada database spasial seperti menterjemahkan query kedalam SQL dan membuat representasi yang diteruskan ke server.

  5. Dalam kenyataannya Komponen Server side GIS berupa software libraries yang menawarkan layanan khusus untuk analisis spasial pada data.

  6. Selain komponen, hal lain yang juga sangat penting adalah aspek fungsional yang terletak di sisi client atau di server.

Web Browser dan Web Page

WWW (World Wide Web) merupakan hypertext based system terdiri atas server dan client komputer menggunakan http yang dapat menyediakan, mengorganisasi, dan mengakses informasi yang mengizinkan pengguna untuk melompat dari satu informasi ke informasi lain. Web dapat bekerja dengan bergantung pada mekanisme di bawah ini.

  1. Hypertext yang menyediakan navigasi yang mudah antar dokumen dan sumber daya.

  2. Protokol merupakan seperangkat standar yang digunakan untuk mengakses sumber daya melalui web.

  3. Universal Resource Locator (URL) merupakan skema penamaan seragam untuk sumber daya Internet

  4. Komputer client dan server.
